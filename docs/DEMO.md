# KnowledgeChat AI Demo Guide

This guide demonstrates the local RAG workflow using the educational Markdown
documents included in `knowledge_base/`. Model wording may vary, but the source
cards should point to relevant repository documents.

## 1. Start Ollama

```bash
ollama serve
```

Keep this terminal running.

## 2. Pull the Models

```bash
ollama pull gemma3
ollama pull nomic-embed-text
```

## 3. Configure and Start the Backend

```bash
cd backend
python -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
cp .env.example .env
uvicorn app.main:app --reload
```

Verify the health endpoint in another terminal:

```bash
curl http://localhost:8000/api/v1/health
```

## 4. Index the Knowledge Base

From the `backend/` directory with the virtual environment active:

```bash
python scripts/index_knowledge.py
```

The script reads the repository Markdown files, generates embeddings through
Ollama, and stores vectors in the local ChromaDB path.

## 5. Start the Frontend

In another terminal:

```bash
cd frontend
npm install
cp .env.example .env
npm run dev
```

Open `http://localhost:5173`.

## 6. Ask a Sample Question

Try one of these questions:

- What is object-oriented programming in Python?
- How does FastAPI routing work?
- What is Docker used for?
- How does Git branching work?
- What is a React component?

## 7. Present the Result

During the demo:

1. Show the user question.
2. Show the Markdown answer generated by the local chat model.
3. Show the source cards beneath the answer.
4. Point out the document path, section, chunk index, and similarity score.
5. Explain that retrieved repository content is added to the model prompt.
6. Explain that Ollama runs locally and no cloud model API is required.

The application does not return full chunk text or embedding vectors to the
browser. Citations contain metadata only.

## Docker Alternative

With Ollama running on the host:

```bash
cp .env.example .env
docker compose up --build
docker compose exec backend python scripts/index_knowledge.py
```

Then open `http://localhost:5173` and follow the same question-and-source flow.
