# ICAP Profilleri

## Kapsam

ICAP Profilleri Nedir?

## Menü yolu

- `ICAP Profilleri > ICAP Server`
- `ICAP Profilleri > Yeni Kayıt`
- `ICAP Profilleri - Yeni Kayıt`

## Kullanım adımları

1. Adı alanına bir ad girin.
2. Request Process kontrolunu seçin veya bırakın.
3. Request Server düşümüne bir sunucu seçin.
4. Request Path alanına istek yolu girin.
5. Request Process On Failure kontrolunu seçin veya bırakın.
6. Response Process kontrolunu seçin veya bırakın.
7. Response Server düşümüne bir sunucu seçin.
8. Response Path alanına cevap yolu girin.
9. Response Process On Failure kontrolunu seçin veya bırakın.
10. Maksimum Bağlantı alanına bağlantı sayısını girin.
11. Overload Option düşümüne bir seçenek seçin.
12. Streaming Media Bypass kontrolünü seçin veya bırakın.
13. Adı alanına serverin adını girin.
14. IP Adresi alanına geçerli bir IPv4 IP adresi girin.
15. Port alanına server portunu girin.
16. Kaydet butonuna tıklayarak kaydınızı kaydedin.

## Alanlar

- `#` (ICAP Profilleri paneli): A unique identifier for each ICAP profile.
- `Adı` (ICAP Profilleri paneli): The name of the ICAP profile.
- `Request Process` (ICAP Profilleri paneli): The process for handling requests.
- `Request Server` (ICAP Profilleri paneli): The server responsible for the request.
- `Request Path` (ICAP Profilleri paneli): The path of the request.
- `Request Process On Failure` (ICAP Profilleri paneli): The process to handle requests when there is a failure.
- `Response Process` (ICAP Profilleri paneli): The process for handling responses.
- `Response Server` (ICAP Profilleri paneli): The server responsible for the response.
- `Response Path` (ICAP Profilleri paneli): The path of the response.
- `Response Process On Failure` (ICAP Profilleri paneli): The process to handle responses when there is a failure.
- `Maksimum Bağlantı` (ICAP Profilleri paneli): The maximum number of connections allowed.
- `Overload Option` (ICAP Profilleri paneli): Overload Option
- `Streaming Media Bypass` (ICAP Profilleri paneli): Streaming Media Bypass
- `IP Adresi` (ICAP Profilleri section): IP Adresi
- `Port` (ICAP Profilleri section): Port
- `İşlemler` (ICAP Profilleri section): Actions field to perform actions on ICAP Profiles or Servers

- `Sayfa Başı Kayıt Sayısı`: Sayfa Başı Kayıt Sayısı

- `Response Process On Failure`: Cevap işlemi başarısız olduğunda davranış seçmek için kullanılır.

- `+ Ekle`: + Ekle

## Uyarılar

- Geçersiz IP adresi! - IP Adresi alanındaki IP adresi geçerli bir IPv4 formatında olmalıdır.
- Adı alanındaki değerlerin eksik olup olmadığını kontrol edin. (Empty field in screenshot, no visible evidence of input required.)
- Request Server ve Response Server düşümlerindeki seçeneklerin eksik olup olmadığını kontrol edin.
- Request Path ve Response Path alanlarındaki değerlerin doğru olduğundan emin olun.
- Model aynı etiketi control ve field olarak sınıflandırdı; yapılandırma değeri olan field korundu: Adı, Request Path, Request Server, Response Path, Response Server
